# Contributing

Contributions, improvements, fixes, and engineering discussions are welcome.

This repository focuses on building production-aware, engineering-focused, interview-useful technical knowledge platforms.

---

# Contribution Philosophy

The goal is to keep repositories:

- Engineering focused
- Easy to understand
- Production aware
- Operationally realistic
- Revision friendly
- Interview useful
- Long-term maintainable

Target feeling:

> Engineer explaining to engineer.

---

# Contribution Guidelines

Please ensure contributions:

- Focus on engineering reasoning
- Include practical production awareness
- Explain tradeoffs clearly
- Avoid unnecessary theory
- Avoid excessive academic explanations
- Avoid generic AI-style filler content
- Keep explanations concise and readable
- Prefer real-world examples where possible
- Keep documentation scalable and maintainable

---

# Recommended Content Style

Strong contributions usually explain:

1. What the technology/system is
2. Why it exists
3. What problems it solves
4. Production tradeoffs
5. Scaling considerations
6. Reliability concerns
7. Operational risks
8. Common failures
9. Debugging approaches
10. Interview discussion angles

---

# Documentation Principles

Preferred writing style:

- Simple engineering language
- Real-world examples
- Troubleshooting oriented
- Production-aware explanations
- Comparison-driven explanations
- Revision-friendly structure
- Practical over theoretical

Avoid:

- Documentation dumps
- Large copied vendor documentation
- Overly academic content
- Definition-only notes
- Excessive filler explanations

---

# Pull Request Expectations

Before submitting changes:

- Ensure workflows pass
- Validate markdown formatting
- Validate YAML formatting
- Keep folder structure clean
- Maintain repository consistency
- Avoid unrelated changes in same PR
